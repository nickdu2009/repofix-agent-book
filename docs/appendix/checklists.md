# 学习检查清单

## Agent MVP

- [ ] `make -C examples/repofix test` 在没有云端密钥时通过。
- [ ] 能画出模型、工具、状态、观察和停止条件的关系。
- [ ] 能手写不依赖 Agent 框架的最小循环。
- [ ] 没有测试通过时，程序会拒绝模型的 `finish` 请求。
- [ ] 测试通过后再次修改代码，程序会拒绝旧测试结果。
- [ ] 最大步数、总时限和取消信号都能终止 Run。
- [ ] 单元测试默认使用 Fake Model，不访问在线 API。

## 工具与安全

- [ ] 相对路径、绝对路径、`..` 和符号链接都经过工作区边界检查。
- [ ] 接入 Daytona 前，真实模型不会执行任何宿主命令或修改后代码。
- [ ] 模型选择可信命令 ID，而不是提交任意 shell 字符串。
- [ ] 输出有长度上限，超时、取消后仍会清理进程和 Sandbox。
- [ ] 沙箱中没有主 GitHub Token 或生产密钥。

## Go 控制平面

- [ ] 能解释 Run 的每一个合法状态转换。
- [ ] Python、Go、TypeScript 的状态和事件通过共享 Schema 验证。
- [ ] 取消、超时、失败和重试不会产生相互矛盾的状态。
- [ ] 服务重启后，未完成 Run 会恢复或明确失败。
- [ ] 事件可重放，浏览器断线后不会丢失最终状态。

## 评测

- [ ] 至少有 3 个可信案例形成 MVP，最终扩展到 10 个以上。
- [ ] 验收由测试或确定性规则完成，不让模型给自己打分。
- [ ] 最终 Oracle 在新的 Sandbox 中运行，Agent 看不到隐藏测试。
- [ ] 能比较 Prompt 或模型修改前后的成功率、成本和步骤数。
- [ ] 能区分 Agent 失败、基础设施失败和案例本身无效。

## TypeScript Web

- [ ] 后端数据以 `unknown` 进入，并在边界做运行时校验。
- [ ] RunEvent 使用统一事件信封和可辨识联合类型。
- [ ] SSE 断线、错误、空数据和加载状态都有明确表现。
- [ ] Reducer 按 `sequence` 去重，关键状态转换有单元测试。

## 最终自测

- [ ] 可以用 10 分钟讲清 RepoFix 的架构和一次 Run 的完整生命周期。
- [ ] 可以独立定位一次失败发生在模型、工具、沙箱还是控制平面。
- [ ] 可以用数据说明一次改进是否真的有效。
- [ ] 可以指出系统当前的安全边界和暂不支持的场景。
