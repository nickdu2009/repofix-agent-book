# 章节实践模板

这本书的每一章都按同一个闭环学习。目的不是“看完”，而是把知识变成可以运行、测试和解释的系统能力。

## 第一步：建立概念模型

阅读章节前，先写下自己的答案：

- 这个能力解决什么工程问题？
- 如果不实现它，RepoFix 会怎样失败？
- 它属于模型决策、程序约束，还是运行基础设施？

## 第二步：完成小练习

先写一个不超过 30 行的独立练习。例如学习 `Protocol` 时，先定义一个 `ModelClient` 和一个 `FakeModelClient`，再把它放入主项目。

## 第三步：亲手实现主路径

以下首个版本建议选择 `Learn Manually`：

- 第一次模型 API 调用
- 第一个 Agent Loop
- Agent State
- Tool Call 执行流程
- Stop Condition
- Go Run 状态机
- 第一个沙箱 Adapter
- 第一个评测运行器

## 第四步：用测试证明行为

至少覆盖一条成功路径和两条失败路径。不要把“程序跑完了”当作验收；要能指出哪条断言证明了行为正确。

## 第五步：让 Codex 审查

```text
Review this branch against main.

重点检查：
- 行为是否满足 Issue 验收条件
- 状态机是否存在遗漏
- 超时和取消是否正确
- 是否存在不安全的命令执行
- 测试是否覆盖失败路径
- 是否出现不必要的抽象

不要修改代码，先输出按严重程度排序的问题。
```

## 第六步：写学习记录

每章结束记录四件事：

1. 我现在能解释什么？
2. 哪个失败案例改变了我的理解？
3. 哪部分仍依赖 Codex 才能完成？
4. 下一章开始前必须解决什么？

只有当你能不看生成过程，解释状态如何流转、为什么调用工具、失败后如何恢复，以及测试如何证明正确时，这个能力才算完成。
