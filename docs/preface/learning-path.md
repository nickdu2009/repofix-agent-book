# 教程目标与学习方法

本教程通过一个有完整工程意义的主线项目，学习 AI Agent 的核心原理、可靠执行、安全隔离、评测和产品化。

主线项目名称：**RepoFix——自主软件缺陷修复 Agent**。

用户提供一个 GitHub 仓库和缺陷描述，RepoFix 自动完成：

```text
理解任务
  → 探索代码仓库
  → 搜索相关代码
  → 制定修复计划
  → 修改文件
  → 运行测试
  → 根据测试结果继续修复
  → 输出 Diff、测试报告、执行轨迹和成本
```

完成本教程后，应具备以下能力：

- 能手写一个不依赖 Agent 框架的 Agent Loop。
- 能设计模型工具、状态、停止条件和失败恢复策略。
- 能用 Go 构建可靠的 Agent 控制平面。
- 能用 Python 实现模型决策、工具调用和评测。
- 能在隔离云沙箱中安全运行 Agent 生成的命令。
- 能用 TypeScript 构建 Agent 运行轨迹与评测界面。
- 能通过评测数据判断 Prompt、模型或工具修改是否有效。
- 能使用 ChatGPT 学习和设计，使用 Codex 加速编码、测试和审查。

## 2. 学习与实施原则

### 2.1 先闭环，后增强

首先只解决一个小型、可验证的缺陷，不追求通用性。最小闭环是：

```text
读取任务 → 读取文件 → 修改文件 → 运行测试 → 确认完成
```

完成最小闭环以后，再逐步加入代码搜索、上下文选择、持久化、重试、评测和 Web 界面。

### 2.2 第一版不使用 LangGraph

第一版手写 Agent Loop 和状态流转，亲自理解：

- 模型如何请求工具。
- 工具结果如何返回模型。
- 上下文如何随步骤增长。
- 系统如何判断成功、失败和停止。
- 模型声明成功与系统验证成功有什么区别。

LangGraph 只作为主项目完成后的可选研究内容，不进入 RepoFix 的关键路径。

### 2.3 模型负责提出动作，程序负责验证动作

模型可以建议写文件、运行测试或者结束任务，但程序必须验证：

- 文件路径是否位于工作区内。
- 命令是否允许执行。
- 参数是否符合 Schema。
- 是否超过步数、时间或费用预算。
- 测试是否真实通过。
- 沙箱是否已经正确清理。

### 2.4 Codespaces 与 Agent 沙箱分离

- **GitHub Codespaces**：供开发者编写、调试和审查代码，可能含有开发密钥。
- **Daytona**：执行 Agent 生成的代码和命令，不注入主 GitHub Token 或生产密钥。

不要把 Codespaces 当作不可信代码的执行沙箱。
