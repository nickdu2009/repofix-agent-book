# 交付门禁设计

| 触发器 | 确定性检查 | 需要云凭据的检查 |
| --- | --- | --- |
| 每次提交 | 单测、格式、类型、契约 | 无 |
| Pull Request | Fake E2E 与构建 | 可选 Sandbox Smoke |
| 手动触发 | 固定案例选择 | 真实模型 Smoke |
| 每周 | 完整离线评测 | 限额真实评测 |

Secret 只由部署或显式工作流注入，日志保存请求 ID 而不保存凭据。回滚以错误率、成功率退化和成本上限为触发条件。

本参考答案定义门禁策略，不表示 Railway 镜像、迁移或真实部署已经完成。
